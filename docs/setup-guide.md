# 설치 및 업로드 순서 가이드

## 1. 압축 해제
1. `BPW-v007-github.zip` 파일을 다운로드합니다.
2. 압축을 풀면 `BPW-v007-github` 폴더가 나옵니다.

## 2. GitHub 저장소 만들기
1. GitHub에 로그인한 뒤 새 저장소(Repository)를 만듭니다.
2. 저장소 이름 예시: `bpw-blog-writer`
3. Public 또는 Private 중 원하는 것을 선택합니다. (개인 테스트용이면 Private 추천)

## 3. GitHub 업로드
1. 새 저장소 페이지에서 "Add file → Upload files"를 클릭합니다.
2. `BPW-v007-github` 폴더 안의 모든 파일/폴더(`index.html`, `assets/`, `worker/`,
   `docs/`, `README.md`, `version-log.md`)를 구조를 유지한 채로 업로드합니다.
3. Commit 버튼을 눌러 저장합니다.

## 4. GitHub Pages 설정
1. 저장소의 **Settings → Pages** 메뉴로 이동합니다.
2. Branch를 `main`, 폴더를 `/ (root)`로 선택하고 저장합니다.
3. 1~2분 기다리면 `https://아이디.github.io/저장소이름/` 주소가 발급됩니다.

## 5. Cloudflare Worker 배포
1. Cloudflare 계정으로 로그인한 뒤 Wrangler CLI를 설치합니다.
2. `worker/wrangler-example.toml`을 복사해서 `wrangler.toml`로 이름을 바꾸고
   본인 환경에 맞게 수정합니다.
3. `worker/worker.js`를 그대로 사용해 `wrangler deploy` 명령으로 배포합니다.
4. 배포가 끝나면 `https://....workers.dev` 형태의 Worker 주소가 발급됩니다.
5. 네이버 검색을 실제로 사용하려면 아래 명령으로 Secret을 등록합니다.
   ```
   wrangler secret put NAVER_CLIENT_ID
   wrangler secret put NAVER_CLIENT_SECRET
   ```
6. AI 글 생성을 실제로 사용하려면 아래 명령으로 Secret을 등록합니다.
   ```
   wrangler secret put AI_API_KEY
   ```
7. Blogger 임시저장/예약발행을 실제로 사용하려면 아래 명령으로 Secret을 등록합니다.
   ```
   wrangler secret put GOOGLE_CLIENT_ID
   wrangler secret put GOOGLE_CLIENT_SECRET
   wrangler secret put GOOGLE_REFRESH_TOKEN
   wrangler secret put BLOGGER_BLOG_ID
   ```
   GOOGLE_REFRESH_TOKEN을 발급받는 절차는 `docs/api-setting-guide.md`를 참고하세요.

(위 Secret을 하나도 등록하지 않아도 앱은 정상 작동하며, 해당 기능은 자동으로
Mock 결과/Mock 저장으로 진행됩니다)

## 6. Worker URL 복사
1. 배포 완료 후 터미널에 출력된 Worker 주소를 복사해둡니다.

## 7. 설정 화면에 Worker URL 입력
1. GitHub Pages로 접속한 앱에서 설정 화면에 Worker URL을 입력한 후, ADMIN_PASSWORD로 로그인합니다.
2. "설정" 탭으로 이동해 "Cloudflare Worker URL" 입력란에 복사한 주소를 붙여넣습니다.
3. "Worker URL 저장" 버튼을 누릅니다.

## 8. Worker 연결 테스트
1. 바로 아래 "Worker 연결 상태" 카드에서 "Worker 연결 테스트" 버튼을 누릅니다.
2. `/health` 경로를 호출해 Worker가 응답하는지 확인합니다.

## 9. 성공 여부 확인
- 성공: "연결 성공" 배지로 바뀌고 동작 모드가 "Worker 모드"로 표시됩니다.
- 실패: "연결 실패, Mock 모드로 계속 사용합니다" 안내가 뜨고, 동작 모드는 자동으로
  "Mock 모드"로 유지됩니다. 이 경우에도 키워드 수집/글 생성/품질검수/Blogger 저장
  등 다른 기능은 평소처럼 Mock으로 계속 사용할 수 있습니다.

## 10. 네이버 검색 / AI 글 생성 실제 연결 확인
1. Worker 연결 테스트가 성공하고 NAVER Secret도 등록되어 있다면, "키워드" 탭에서
   실제 블로그/웹문서 결과가 표시됩니다. 등록되어 있지 않으면 자동으로 Mock 결과로
   전환되고 "네이버 검색 연결 실패, Mock 결과로 계속 진행합니다." 안내가 표시됩니다.
2. AI_API_KEY도 등록되어 있다면, "글생성" 탭에서 "실제 AI 글 생성" 배지와 함께 AI가
   작성한 글이 표시됩니다. 등록되어 있지 않으면 "Mock 글 생성" 배지와 함께 자동으로
   Mock 글로 전환됩니다.

## 11. Blogger 실제 연결/저장 확인
1. Worker 연결 테스트가 성공하고 Blogger Secret(`GOOGLE_CLIENT_ID`,
   `GOOGLE_CLIENT_SECRET`, `GOOGLE_REFRESH_TOKEN`, `BLOGGER_BLOG_ID`)이 모두 등록되어
   있다면, "발행" 탭의 "Blogger 연결하기" 클릭 시 "연결됨 (실제 Blogger)"로 표시됩니다.
2. Secret이 없거나 연결 확인(`/blogger/status`)이 실패하면, 자동으로 "연결됨 (Mock)"으로
   전환되고 실패 사유가 안내됩니다.
3. 임시저장/예약발행 버튼을 누르면 실제 연결인 경우 `/blogger/draft` 또는
   `/blogger/schedule`을 시도합니다. 실패하면 자동으로 Mock 저장으로 대체되고, 저장
   목록에는 "실제 Blogger" 또는 "Mock" 배지로 구분되어 표시됩니다.
4. ⚠️ 바로 발행 기능은 없습니다. 임시저장 후 Blogger 관리자 화면에서 최종 확인하거나,
   예약발행으로 등록한 시간에 자동 발행되도록 운영하세요.

## 12. 하루 저장/예약 제한 설정
1. "설정" 탭의 "하루 저장/예약 제한" 입력칸에서 1~10 사이 값을 입력하고
   "하루 제한 저장"을 누릅니다. (기본값 3건, 초기 운영 권장값 3~5건)
2. 대시보드와 "발행" 탭 모두 이 설정값을 기준으로 오늘 저장/예약 가능 건수를 계산합니다.
3. 오늘 저장/예약한 건수가 설정값에 도달하면, 추가 임시저장/예약발행 버튼이
   비활성화되고 "오늘 발행 제한 N건을 초과했습니다" 안내가 표시됩니다.

## 13. 생성 모드 / 글쓰기 옵션 / 생성 취소
1. "글생성" 탭에서 빠른 생성(800~1000자) / 일반 생성(1200~1500자) / 고급 생성(2000자
   이상) 중 하나를 선택합니다. 참고자료 개수, FAQ/체크리스트 개수도 모드에 따라
   달라집니다.
2. 글쓴이 느낌(중립/남성 느낌/여성 느낌), 작성 톤(기본형/친근한 블로그형/꼼꼼한
   리뷰형/생활 꿀팁형/전문가 정리형), 이모티콘 사용(사용 안 함/적게 사용/적당히 사용)을
   선택할 수 있습니다. 기본값은 각각 "중립", "친근한 블로그형", "적게 사용"입니다.
3. "글 생성하기"를 누르면 버튼이 "AI 글 생성 중..."으로 바뀌고, "최신정보를 분석하고
   있습니다." 같은 진행 문구가 순환 표시됩니다.
4. 생성 중 "생성 취소" 버튼을 누르면 진행 중인 요청이 즉시 취소되고, 다시 "글
   생성하기"를 눌러 처음부터 생성할 수 있습니다.
5. 생성 중 하단 메뉴(홈/키워드/재료 등)를 누르면 "글 생성 중입니다. 이동하면 결과가
   반영되지 않을 수 있습니다. 이동할까요?" 확인 창이 뜹니다. 취소를 누르면 현재
   화면이 유지됩니다.

## 14. 아침 키워드 브리핑
1. 대시보드의 "브리핑 바로가기" 또는 직접 화면 이동으로 "아침 키워드 브리핑" 화면에
   들어갑니다.
2. "오늘의 트렌드 수집"을 누르면 최신 트렌드/오늘의 이슈/수익형 키워드/글감 추천/
   민감·주의 키워드가 카테고리별로 표시됩니다. (현재는 예시 데이터 기반 Mock 기능)
3. "추천" 태그가 붙은 키워드 카드를 누르면 키워드 수집 화면으로 이동하면서 키워드
   입력칸이 자동으로 채워집니다. "주의" 태그가 붙은 항목(정치/사건사고/연예 논란 등)은
   클릭해도 이동하지 않습니다.
4. 자동 예약 실행(매일 아침 자동 수집)은 아직 구현되어 있지 않으며, 이후 버전에서
   Cloudflare Worker 예약 실행으로 확장할 수 있도록 구조만 준비되어 있습니다.

## 15. 클릭 테스트 순서
1. 로그인 → 대시보드의 연결 상태 카드(Worker/네이버/AI/Blogger) 확인
2. Worker URL 저장 및 연결 테스트
3. 하루 저장/예약 제한 값 변경 및 저장 확인
4. 아침 키워드 브리핑 → 트렌드 수집 → 추천 키워드 클릭 시 키워드 화면 이동 확인
5. 키워드 입력 → 최신정보 수집(실제 또는 Mock) 확인
6. 나만의 글 재료 입력 → 저장 확인
7. 생성 모드/글쓰기 옵션 선택 → 글 생성 → 진행 문구·생성 취소·이동 경고 확인
8. 미리보기 → 상단 X 버튼/하단 닫기 버튼으로 복귀 확인
9. 품질검수 → 점수, 부족한 항목 카드, "보완 후 다시 생성" 버튼 확인
10. Blogger 연결(실제 또는 Mock) → 연결 전에는 저장 버튼이 비활성화되는지 확인
11. 임시저장(70점 이상) / 예약발행(85점 이상 + 날짜·시간 입력) 확인, 저장 목록의
    저장 방식(실제 Blogger/Mock)·postId·링크 표시 확인
12. 설정 화면에서 글 톤, 금지어, 전체 초기화 확인

## 16. 주의사항
- 이 저장소(특히 Public 저장소)에는 실제 API 키, OAuth 비밀값, 실제 비밀번호를
  절대 올리지 마세요.
- 실수로 올렸다면 즉시 키를 재발급하고, Git 히스토리에서도 제거해야 합니다.
